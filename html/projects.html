Clippy

Overview

GitHub For Clippy

Clippy is essentially an all-in-one slack app. The "all" in this may lead one to believe that it has a lot of useful features, but this is definitely not the goal with the app. The current goal is to set up a robust framework for an app so my friends and I can set up any features we want easily. If we happen to make some useful features, so be it.

The implementation is pretty simple using the Slack API. The application is running on a Digital Ocean server (in New York or something like that). It communicates with Slack through HTTP Post requests pretty much entirely, so dealing with different types of events on the Slack group is pretty easy.

Current State

Initially, Clippy started out as a Node.js application running on the same instance as the website you're on! With this, I was able to write a couple features for Clippy in javascript:

  • Clippy Weather: This is a super simple command which just responds with the current weather in Pittsburgh when someone says "clippy weather"
  • Clippy Groceries: This is a slightly more intensive feature which keeps track of a group grocery list with adding, viewing and deleting capabilites. The data is stored via MySQL on the same server as the app.

However, because of my goal with letting others contribute, I decided that javascript wasn't the best language to use to implement the rest of the app in. I pretty quickly settled on Python, and in my stubborness to not want to switch from using Node.js, I initially went with an approach of trying to integrate Node.js with python on this app. This proved to be a stupid idea, as the only thing I was using node for could easily be done using Flask and Requests in Python. So, for now, I am working on moving all of Clippy to a Python App, detaching it from my website entirely.

Future

Once I'm done moving to Python, I plan on adding a few new features/fixes:

  • Improved Regular Expressions: While Slack has interactive messages, I find it quite limiting (especially since I can't render any of my own HTML for security reasons on their part). For this reason, we decided it would be best to lean into this limitation and simply make a robust text bot. Naturally, this requires decent regular expression work to get a decent one working, and currently the regular expressions I'm using are quite basic, so more would be nice.
  • User Predictor: Try and use the 10k messages saved by Slack to train a classifier to predict who a user is based on the message sent. The data is quite easy to get, but I haven't settled on a specific ML architecture/method to use in this. Linear SVC or Naive Bayes seem like the ways to go. I've implemented Naive Bayes for a class project before so I might just stick with that.
  • Mark's Jumper: For a more fun feature, my (only) Irish friend in Carnegie Mellon often responds to questions of the form: "Where is ____" with "Up me jumper". I think it'd be nice to keep track of what's up there, so this would essentially be a renamed version of Clippy Groceries.
